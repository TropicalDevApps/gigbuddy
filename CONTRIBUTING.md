# Contributing to gigBuddy

Thank you for your interest in contributing to gigBuddy!

## How to Contribute

1.  **Fork the repository.**
2.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make your changes.** Ensure your code follows the existing style and passes all tests.
4.  **Commit your changes.** Use clear and descriptive commit messages.
5.  **Push to your fork:**
    ```bash
    git push origin feature/my-new-feature
    ```
6.  **Create a Pull Request.**

## Standards

-   Use `npm run lint` to ensure code quality.
-   Ensure the project builds correctly with `npm run build`.
-   Respect the project's branding and design principles.
